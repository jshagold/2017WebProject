최상단 타이틀: 해당 타이틀을 클릭시 메인메뉴(매칭, 대기줄, 정보)로 이동합니다.

사이드바: 왼쪽 상단의 사람모양 아이콘을 누르면 왼쪽에서 메뉴가 슬라이드 되면서 나오며 기존의 화면은 어둡게 변합니다.
	사이드바안에는 사용자사진, 사용자이름, 사용자이메일 정보가 담겨있으며 제일 하단에는 로그아웃 버튼이 있습니다.


--> 최상단 타이틀과 사이드바는 모든 메뉴에서 항상 같은 곳에 위치하고 있습니다.

로그인: 구글 계정으로 로그인을 통해서 접속자의 구글 계정으로 로그인을 할 수 있습니다.

메인: 메인은 총 3개의 버튼이 상하로 (매칭, 대기줄, 정보)로 구성되어있습니다.
	매칭을 누를 경우 --> 대기줄, 정보 버튼이 작아지며 아래로 내려가 사라지게 됩니다.
	그리고 매칭작성을 할 수 있는 창이 뜹니다. 매칭 작성창은 하나 입력을 완료해야 다음 입력을 할 수있게 구성되어 있으며, 
	모든 입력을 완료해야 최종적으로 매칭을 하는 버튼이 출현합니다.
	그리고 빠른매칭, 예약매칭 각각의 경우 출현하는 입력창이 다릅니다.

	-매칭 작성창 구성
	--빠른매칭, 예약매칭 선택
	--음식종류 선택(한식, 분식, 일식, 중식 ...)
	--음식점 선택(각 음식종류에 맞는 식당 등장, 상관없음 옵션도 있습니다.)
	--최대인원 선택(2~6명)

	++[예약매칭에만 있는 구성]
		--날짜 선택(달력이 등장하여 날짜를 선택할 수 있습니다.), 시간 선택
		--코멘트 작성란(짧게 남기고 싶은 말을 남길 수 있는 공란이 등장하는데, 아무것도 입력하지 않을 수도 있습니다.)

	--매칭 버튼 출현

	대기줄 누를시-> 바로 링크로 이동이 됩니다.
	정보 누를시-> 정보버튼이 작아지며 하단에 인증정보, 비인증정보 버튼이 출현합니다. 각 인증정보는 링크로 되어있습니다.

대기줄:	대기줄을 누를 경우 --> 바로 링크로 이동하게 됩니다.
	예약매칭을 통해 등록이 된 사람들의 글을 볼 수 있습니다.
	상단의 음식 종류 선택 메뉴를 통해 글을 구별할 수 있습니다.
	예를들어 무관, 한식, 중식이 있을경우 무관을 선택하면 종류에 상관없이 모든 예약글이 출현하고, 한식을 선택하면 예약글 중 음식종류가
	한식인 글들만 출현하게 됩니다.
	각각의 해당글에는 음식종류, 음식점, (참여인원/참여인원제한), 만나는 시간 정보(날짜, 시간)가 뜹니다. 
	그리고 우측에 보기 버튼이 있는데 보기 버튼을 눌렀을 때 해당 글 밑에 글 작성자가 작성한 코멘트를 볼 수 있습니다.
	기존의 보기 버튼은 '보기'라는 글자가 '참가'로 변경이 되고 이 후에 그 버튼을 누르게 되면 그 예약에 참가하게 되는 참가버튼으로 바뀝니다.
	해당 예약글의 코멘트를 그만보고 싶거나 참가를 희망하지 않는다면 코멘트 밑에 등장하는 닫기 버튼을 클릭하면 기존의 상태로 돌아가게 됩니다.

정보: 	정보를 누를 경우 --> 정보버튼이 작아지며 하단에 인증정보, 비인증정보 버튼이 출현합니다. 인증정보, 비인증정보는 링크로 이동하게 됩니다.
	정보메뉴를 통해서 사람들이 직접 작성한 음식점, 음식 정보 등을 볼 수 있으며 본인도 작성할 수 있습니다.
	음식점, 음식종류, 주 판매 메뉴, 가격을 볼 수 있으며, 해당 글을 클릭하게되면 글 아래로 작성자가 작성한 세부평을 볼 수 있습니다.
	인증정보와 다르게 비인증정보는 세부평 하단에 추천하기 버튼이 있습니다. (인증정보는 일정 수 이상의 추천을 받은 정보만 보여줍니다.)
	인증정보, 비인증정보 각각의 화면 제일 하단에는 글 작성하기 버튼이 있습니다. 해당 버튼을 클릭시 글 작성 페이지로 이동하게 됩니다.

	-글 작성 페이지
	본인이 비인증정보에 올라가게 될 글을 작성 할 수 있습니다.
	식당이름, 음식종류, 주 메뉴 이름, 가격 그리고 추가적인 세부평을 받고 글 작성 완료 버튼을 누를 시 글이 등록이 되게 됩니다.
